Cómo actualizar los datos

Opción 1 — Automático (recomendado):

GitHub Actions actualiza los datos automáticamente todos los días a las 6am UTC.


Opción 2 — Manual desde GitHub:

  1. Ir a tu repositorio en GitHub.com
  2. Hacer clic en la pestaña Actions
  3. Elegir el workflow "Actualizar Datos de Arqueros"
  4. Hacer clic en Run workflow → Run workflow
  5. Esperar ~5-10 minutos que termine
  6. Recargar esta página

Opción 3 — Local (si tenés Node.js instalado):

cd scripts && npm install && node scrape.js && node build-data.js

¿Cómo funciona la segmentación?


Para vender un dashboard personalizado a un arquero o club, se necesita:

  1. Identificar el archer_id o club_id del cliente (usando el buscador arriba)
  2. Ejecutar el script de segmentación: node scripts/segment.js --archer {id} --output ./builds/{nombre}/
  3. El script genera un paquete de JSON que contiene solo los datos de ese cliente
  4. Ese paquete se sube a un repositorio GitHub separado con su propio GitHub Pages

⚠ Los filtros del dashboard principal NO son privacidad real — cualquiera puede ver todos los datos. La segmentación real requiere generar paquetes separados.
